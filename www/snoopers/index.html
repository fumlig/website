SSH Snoopers Around the World

2022-02-27

I run this website on a cheap machine from my home. It has been live on and off since around April 2021. I use SSH to administrate the machine and have opened the SSH port to the public.

Being able to access the machine from everywhere is extremely convenient, but it also brings some unwanted attention. I tend to get many malicious connections from around the world every day. These are likely performed by some script that tries common usernames and passwords on all open SSH servers with hopes to get access to the server and some valuable information. There are many exposed SSH servers with guessable usernames and passwords, so it is likely not a fruitless pursuit.

In this article I thought I would take a closer look at the login attempts made to my exposed machine. I will redact all IP addresses for privacy purposes.

Collection

My server is accessible on my local network as oskar-server. It runs OpenSSH, whose server daemon sshd is managed with systemd. Its logs can be collected with journalctl.

!host=oskar_server ssh $host journalctl --unit sshd --output short-iso > sshd.log

Let's have a look at some log entries.

The first line above shows what a failed attempt looks like. Someone tried signing in as the user simon on port 39058. We can collect all log entries for failed attempts with a regular expression.

import datetime as dt
import re

ssh = []

with open("sshd.log") as f:
    for line in f:
        match = re.match(r"^(?P<datetime>\S+) (?P<hostname>\S+) sshd\[\d+\]: Invalid user (?P<user>\S+) from (?P<addr>\S+) port (?P<port>\d+)$", line)
        if match is not None:
            groups = match.groupdict()
            groups["datetime"] = dt.datetime.strptime(groups["datetime"], "%Y-%m-%dT%H:%M:%S%z")
            ssh.append(groups)

Let's store the attempts in a dataframe for convenience.

import pandas as pd

df_ssh = pd.DataFrame(ssh)

Analysis

Now we can have analyse the login attempts. Let's start with the most common usernames.

import matplotlib as mpl

<AxesSubplot:xlabel='username', ylabel='attempts'>

Among the more common usernames are admin (which I assume many people name their superuser), ubnt (apparently the default username for some hardware of an internet service provider named UniFi) and pi (the default username on the Raspbian operating system).

Given that you have someone's IP address, you can also get a rough estimate of where they are. One database of IPs and their locations is GeoIP2, which has a nice Python API. Let's see from where the SSH login attempts are made.

from geoip import geolite2

ips = []

for ip in df_ssh.addr.unique():
    match = geolite2.lookup(ip)
    if match is not None:
        ips.append(match.to_dict())    

df_ip = pd.DataFrame(ips)

Apparently most attempts are from China, the United States, and Vietnam. Of course some people may also be hiding behind a VPN.

The GeoIP database also includes coordinates of the IP addresses which means that we can draw a heatmap of exact locations. Matplotlib Basemap Toolkit has a nice API for plotting 2D data on maps.

from mpl_toolkits.basemap import Basemap

earth = Basemap(projection="merc", llcrnrlat=-80, urcrnrlat=80, llcrnrlon=-180, urcrnrlon=180, lat_ts=20, resolution="c")
earth.drawcoastlines()
earth.fillcontinents(color="olive", lake_color="aqua")
earth.drawmapboundary(fill_color="lightseagreen")
earth.drawcountries()

df_coords = pd.DataFrame(df_ip.location.tolist(), columns=["lon", "lat"]).dropna()
x, y = earth(df_coords.lat, df_coords.lon)

Protection

To protect your SSH server from simple attacks, you should probably:

  • Use public key authentication.
  • Set up two factor authentication.
  • Use some jail like fail2ban to block IP addresses that fail authentication too many times.
  • Disallow root login.
